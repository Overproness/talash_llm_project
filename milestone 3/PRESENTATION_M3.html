Functional Module 3 · 6 Marks

Topic Variability & Co-Author Analysis

Shannon Entropy — Topic Diversity

H = −∑ pᵢ · log₂(pᵢ)
EntropyClassification
H ≥ 2.5Highly Diverse
1.5 ≤ H < 2.5Moderately Diverse
H < 1.5Focused / Niche

Tokens extracted from publication titles; stop-words removed

Co-Author Collaboration

  • Unique collaborators — distinct co-author count across all papers
  • Network density — unique co-authors ÷ total author-slots
  • Repeat collaborators — co-authors in 3+ papers (sustained groups)
  • Solo publication rate — fraction of sole-authored papers
  • Avg. authors per paper — team size trend indicator

Combined into:

collaboration_score (0–100)
Functional Module 4 · 5 Marks

Supervision, Patents & Books

🎓

Student Supervision

  • PhD supervision: +8 pts
  • MS supervision: +4 pts
  • Co-supervision: 50% weight
  • Fields: student name, degree, thesis title, year, role
⚙️

Patents

  • Title, number, country, inventors, date
  • US/EP/WO patents weighted higher
  • Adds to academic impact score
📚

Books & Chapters

  • Sole-authored textbook (Springer/Wiley): highest weight
  • Book chapter in edited volume: medium weight
  • Self-published: low weight
  • Publisher verified via keyword taxonomy

All three supplementary academic outputs are extracted by LLM with rule-based fallback, stored in MongoDB as structured Supervision, Patent, and Book Pydantic models, and surface in the candidate detail page.

Extra Credit · Candidate Ranking Module

Quantifiable Candidate Ranking

Weighted Scoring Formula

Overall = Σ(wᵢ · sᵢ) / Σ(wᵢ)

Education → 30%
Research → 30%
Experience → 25%
Skills → 5%
Completeness → 10%

Denominator normalizes over computed components only — no unfair penalty for CVs lacking certain sections

🏆 Ranking UI

  • Podium view — Gold / Silver / Bronze medals for top 3
  • Full ranked table — per-component score breakdown bars
  • Min-score filter — slider to set minimum threshold
  • Limit control — show top N candidates

API Endpoint

GET /api/candidates/rank
  ?limit=50&min_score=0

→ sorted by overall_score DESC
→ includes rank position (1-indexed)
